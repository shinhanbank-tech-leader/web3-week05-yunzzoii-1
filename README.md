## 레포트 내용

EOA는 블록체인 네트워크에서 사용자가 직접 생성하고 제어하는 가장 기본적인 형태의 계정으로, 암호 키 쌍(개인키, 공개키)로 이루어져 있다. EOA는 개인키를 직접 관리하고, 키를 분실하면 자산 복구가 절대 불가능하다. 또한, 가스비 지불을 위해 해당 체인의 기본 코인을 계정에 가지고 있어야 한다. 

ERC-4337은 블록체인의 트릴레마(보안성, 탈중앙화, 확장성) 을 해결하기 위해 스마트 컨트랙트 기반으로 만든 것이다. 사용자의 지갑을 프로그래밍할 수 있게 만드는 기술이다.

사용성이 좋아질수록, 시스템의 복잡도는 올라가기 때문에 추가적인 규칙, 권한 및 책임 소재를 등을 어떻게 정의하고 안전하게 배포할 것인지에 대해 고민해야 한다. 

- 은행의 인증·승인·계정 관리 방식과 비교했을 때, 비슷한 점과 다른 점은 무엇인가
| 구분 | 전통 은행 시스템 | ERC-4337 기반 지갑 |
| :--- | :--- | :--- |
| **인증** | ID/PW, 인증서, 생체인증 (중앙 서버 검증) | 개인키 서명 (스마트 컨트랙트 코드 검증) |
| **승인** | 내부 결재 라인 및 FDS 통제 | 컨트랙트 내 프로그래밍된 승인 로직 |
| **복구** | 신분 확인 후 DB 값 재설정 | 가디언(소셜 복구)을 통한 컨트랙트 소유권 이전 |
| **차이점** | **중앙화된 원장** 수정 가능 | **탈중앙화 원장** 위에서 코드에 의한 자동 집행 |

- 본인이 생각했을 때, 금융회사 IT 관점에서 가장 현실적으로 받아들일 수 있는 구조는 무엇인가
1. 계층형 인프라 설계(L2중심)
L1(Settlement Layer) > L2(Execution Layer) > App Layer (고객이 사용하는 뱅킹 앱)
2. ERC-4337 기반 스마트 계정 구조
하이브리드 키 관리, 가스비 대납, 정책 등의 핵심 기능 포함.
