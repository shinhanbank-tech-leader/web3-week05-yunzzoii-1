아래 질문을 참고하면 작성에 도움이 됩니다.

- EOA(개인키로 직접 사용하는 기본 계정)는 왜 불편하거나 위험할 수 있는가
- ERC-4337 기반 지갑은 기존 EOA와 무엇이 다른가
- 사용성이 좋아질수록 운영·통제 측면에서는 무엇을 새로 고민해야 하는가
  (예: 승인 규칙, 복구 권한, 수수료 부담 주체, 장애 시 책임)
- 은행의 인증·승인·계정 관리 방식과 비교했을 때, 비슷한 점과 다른 점은 무엇인가
- 블록체인 트릴레마란 무엇이며, 왜 이 문제가 중요한가
- L2(레이어2)는 트릴레마를 해결하기 위해 어떤 방향으로 등장했는가
- 본인이 생각했을 때, 금융회사 IT 관점에서 가장 현실적으로 받아들일 수 있는 구조는 무엇인가
